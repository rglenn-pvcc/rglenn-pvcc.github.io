Operating System Terminology
Term Definition
operating system software that controls the overall operation of the computer includind file management, the user interface, and the program execution environment
job queue list of tasks to be carried out in turn by an operating system
FIFO First In, First Out, a method for handling data structures where the first element is processed first and the newest element is processed last
interactive processing the person needs to provide the computer with instructions whilst it is doing the processing
time-sharing sharing of a computing resource among many users at the same time via multiprogramming and multi-tasking
multiprogramming time is divided into intervals and then the execution of each job is restricted to only one interval at a time
application software consists of the programs for performing tasks particular to the machine's utilization
system software performs tasks that are common to computer systems in general, providing infrastructure
utlitity software consists of software units that extend (or perhaps customize) the capabilities of the operating system
graphical user interface(GUI) interface in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icon
shell text based interface, all actions are performed through text commands
kernel internal part of the operating system, contains those software components that perform the very basic functions required by the computer installation
device driver a component of the kernel, handles communication with the machine's peripheral devices
file manager maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files
memory manager coordinating the machine's use of main memory
virtual memory is the imaginary memory space whose apparent pres- ence is created by the process of swapping data and programs back and forth between main memory and mass storage
boot strapping also called 'booting', procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory
ROM Read Only Memory, nonvolatile memory containing permanent or semi-perminent data
deadlock condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another
spooling technique for granting several processes access to a common resourc
super user (or administrator) the user highly privileged access to the operating system
multi-factor authentication authentication method that requires more than one piece of evidence to gain access
