Chapter 7 Software Engineering
Term Definition
software engineering: the process of developing, testing and deploying computer applications to solve real-world problems by adhering to a set of engineering principles and best practices.
integrated development environment (IDE) a software application that provides comprehensive facilities to computer programmers for software development.

    Data relationships:
  • one-to-one: type of cardinality that refers to the relationship between two entities. A and B in which one element of A may only be linked to one element of B, and vice versa.
  • one-to-many: when a parent record in one table can potentially reference several child records in another table.
  • many-to-many: when multiple records in a table are associated with multiple records in another table.

    Software testing:
  • glass box testing: shows which parts of the program under test have, or have not, been executed. .
  • black box testing: A method of software testing that examines the functionality of an application without peering into its internal structures or workings.
  • beta testing: is a collection which is unordered, unchangeable*, and unindexed. *set items are unchangeable but you can remove items and add new items.
  • alpha testing: the initial phase of validating whether a new product will perform as expected.

    Documentation:
  • user documentation: any form of documentation intended for the end-user of a product or a service.
  • system documentation: documents that describe the system itself and its parts.
  • technical documentation: any document that describes the microservice architecture or use of a software product.

    Intellectual property law:
  • software license: a document that states the rights of the developer and user of a piece of software.
  • software disclaimer: is the software developer's formal notice to the user of its software that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties.
  • best defense against software liability: to apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors.
